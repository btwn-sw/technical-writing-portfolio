# HiDoc — Hybrid Documentation

> AI가 컴플라이언스 문서 초안을 생성합니다. 사람이 검토하고, 수정하고, 승인합니다.

[English →](../../README.md)

<br>

## 문제

컴플라이언스 문서화에는 두 가지 구조적 문제가 있습니다.

첫째, 같은 의무가 여러 규제에 중복으로 등장합니다. GDPR Art.28, ISMS-P, SOC 2는 모두 데이터 처리 계약을 요구하지만, 대부분의 도구는 규제마다 별도의 문서를 생성합니다. 팀은 중복되고 종종 상충하는 내용을 각각 관리해야 합니다.

둘째, 컴플라이언스 문서는 법적 정밀도를 요구하지만 처음부터 작성하려면 대부분의 팀이 갖추지 못한 전문 지식이 필요합니다. 결과적으로 위험할 정도로 형식적인 문서가 만들어지거나, 외부 전문가에게 과도한 비용을 지불하게 됩니다.

<br>

## HiDoc이 하는 일

HiDoc은 회사의 제품 프로파일과 선택한 규제를 입력받아 단일 통합 컴플라이언스 문서를 생성합니다. 모든 섹션은 해당 섹션이 충족하는 규제 조항이 태그로 표시됩니다.

```
입력:  회사 프로파일 + 선택한 규제 (예: GDPR + ISMS-P)
엔진:  원자(atom) 기반 컴플라이언스 매핑
출력:  단일 통합 문서 — 각 섹션에 충족 규제 조항 태그 포함
```

한 섹션이 GDPR Art.13과 ISMS-P를 동시에 충족하면, 그 섹션은 한 번만 생성됩니다. 두 문서에 중복 작성되지 않습니다.

<br>

## 작동 방식

### 원자(Atom) 모델

HiDoc은 각 규제 의무를 원자(atom)로 분해합니다. 원자는 더 이상 나눌 수 없는 최소 단위의 컴플라이언스 요구사항입니다. 각 원자는 다음 정보를 담습니다.

- 출처 규제 조항
- 의무 유형 (`DISCLOSE`, `IMPLEMENT`, `MAINTAIN`, `PERFORM`, `VERIFY`, `REPORT`, `CONTRACT`)
- 적용 조건 (항상 적용 또는 회사 프로파일 기반 조건부 적용)
- 기여하는 출력물

**현재 원자 커버리지:** GDPR 79개 원자 (Art.5-Art.33) + 한국 법정 보존 의무 5개 원자

### 출력 유형

원자는 여섯 가지 출력 유형에 매핑됩니다. 각 유형은 서로 다른 컴플라이언스 기능을 담당합니다.

| 접두사 | 유형 | 목적 |
|---|---|---|
| `DOC_` | 문서 섹션 | 정보 공개 의무 충족 |
| `EVD_` | 증거 산출물 | 감사 제출용 기록 |
| `OPR_` | 운영 절차 | 수행해야 하는 프로세스 |
| `TCH_` | 기술적 통제 | 구현해야 하는 기술 요건 |
| `CTR_` | 계약 산출물 | 체결해야 하는 계약 문서 |
| `POL_` | 정책 기술서 | 공식 정책 선언 |

### 생성 엔진

생성 엔진은 선택한 규제에 매핑된 원자를 조회하고, 회사 프로파일 조건을 적용하며, 주입 순서에 따라 섹션을 정렬한 뒤 Server-Sent Events(SSE)로 클라이언트에 스트리밍합니다.

**생성 엔진 제어 원칙:**
- **Information Discipline** — 원자 매핑에서 명시적으로 허가한 내용만 주입합니다.
- **Mapping Completeness** — 소스 원자가 완전히 매핑된 섹션만 생성합니다.
- **Generation Traceability** — 생성된 모든 섹션은 소스 원자로 역추적할 수 있습니다.
- **`[INFORMATION NEEDED]` 프로토콜** — 회사 프로파일 정보가 없으면 추측하지 않고 명시적인 플레이스홀더로 표시합니다.

→ [아키텍처 상세](architecture.ko.md) · [생성 흐름 상세](how-it-works.ko.md)

## 현재 상태

Phase 1 MVP 완료.

- GDPR Art.5-Art.33: 79개 원자 전체 매핑 완성
- `compliance_outputs` 테이블: 97개 출력 정의
- `atom_output_mapping` 테이블: 147개 원자-출력물 매핑
- 생성 엔진 SSE 스트리밍 구현 완료
- Privacy Policy (9개 섹션, 47개 원자): 첫 완전 생성 검증 완료

**다음 단계:** EU AI Act 원자 구축 · DPA 생성 엔진 추가 · Company Profile 입력 UI 구현

## 내 역할

**창업자 — 제품 설계, 아키텍처, AI 활용 개발**

HiDoc은 의료기기 산업에서 30개국 이상의 컴플라이언스 문서화를 8년간 담당한 경험에서 출발했습니다. 원자 모델, 4레이어 아키텍처, 출력 유형 체계, 생성 엔진 제어 원칙은 모두 직접 설계했습니다. 프로그래머가 아니기 때문에, 구현 전 과정은 Claude를 개발 파트너로 활용해 AI 보조 방식으로 진행했습니다.

항상 필요했지만 존재하지 않았던 도구를 만드는 것이 목표였습니다.
규제 구조를 충분히 싶이 이해해, 통합적이고 추적 가능하며 감사에 통과할 수 있는 문서를 설계 단계부터 생성하는 도구입니다.
