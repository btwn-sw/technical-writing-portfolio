# 개발자 포털 — 정보 구조 설계

<br>

## 이 문서에서 얻는 것

플랫폼이 특화된 API를 외부 개발자에게 처음 공개할 때, 개발자 포털의 구조가 개발자 스스로 연동에 성공할지, 아니면 결국 서포트 티켓을 남기게 될지를 가릅니다. 이 케이스 스터디는 하나의 개발자 포털 정보 구조를 설계하며 거친 판단 과정을, 독자 분석부터 최종 사이트맵까지 다룹니다.

[English →](../en/README.md)

<br>

## 상황

물류 플랫폼이 실시간 배송 추적 API를 소매업체, 풀필먼트 파트너, 연동 벤더 같은 외부 개발자에게 처음으로 공개합니다. 이 플랫폼은 경쟁사가 갖지 못한 데이터, 즉 자사 네트워크 안에서 생성되는 실시간 배송·배달 신호를 보유하고 있습니다. 이 데이터는 개발자 포털이 이를 쉽게 찾고 쓸 수 있도록 구조화되어 있을 때만 실제 강점이 됩니다.

<br>
